# local-terminal-mcp

Give Claude controlled access to your local Windows machine ‚Äî browse files, read code, run approved commands, and manage projects without leaving your AI workflow.

Runs as a Windows Service so Claude stays connected across sessions. All commands are audited. Destructive patterns are hard-blocked server-side.

---

## What it does

Claude gets seven tools across three safety tiers:

**Tier 1 ‚Äî Read-only (always safe)**
- `list_directory` ‚Äî list files and folders
- `read_file` ‚Äî read up to 500 lines of any text file
- `get_system_info` ‚Äî OS version, disk space, memory, running processes
- `find_files` ‚Äî search for files by name pattern

**Tier 2 ‚Äî Approved commands**
- `run_npm_command` ‚Äî `install`, `ci`, `list`, and `run <script>` only
- `run_git_command` ‚Äî read-only git: `status`, `log`, `diff`, `branch`, `fetch`

**Tier 3 ‚Äî Escape hatch**
- `run_command` ‚Äî arbitrary shell command. `dry_run=true` by default; Claude must preview before executing. Hard-blocked patterns are enforced server-side regardless of input.

### Hard-blocked patterns
`rm -rf`, disk format, `del /s`, shutdown, `reg delete`, `cipher /w`, `bcdedit`, `diskpart`, and `sc stop/delete local-terminal-mcp`.

## Requirements

- Windows 10 / 11
- [Node.js](https://nodejs.org) v18 or later
- PowerShell (run as Administrator)

## Installation

```powershell
# 1. Clone or download the repo
git clone https://github.com/claudedussy/local-terminal-mcp
cd local-terminal-mcp

# 2. Run the installer as Administrator
.\setup.ps1
```

`setup.ps1` will:
- Build the project
- Generate a random auth token and save it to `.env`
- Download NSSM and install `local-terminal-mcp` as a Windows Service
- Configure auto-restart on crash (3s delay)
- Print the `claude_desktop_config.json` snippet to paste

Add the printed snippet to your Claude Desktop config, then restart Claude Desktop.

**Config file location:**
```
%LOCALAPPDATA%\Packages\Claude_pzs8sxrjxfjjc\LocalCache\Roaming\Claude\claude_desktop_config.json
```

---

## Updating

```powershell
git pull
.\setup.ps1
```

Re-running `setup.ps1` stops and removes the existing service, installs the new version, and restarts ‚Äî your `.env` (auth token) is preserved.

---

## Uninstalling

```powershell
.\uninstall.ps1
```

Stops and removes the service, prompts before deleting the install directory, and reminds you to clean up `claude_desktop_config.json`.

## Configuration

All settings live in `.env` (auto-generated by `setup.ps1`):

| Variable | Default | Description |
|---|---|---|
| `MCP_AUTH_TOKEN` | auto-generated | Bearer token required on every request |
| `MCP_PORT` | `3002` | Local port (localhost-only ‚Äî not reachable from network) |
| `MCP_LOG_DIR` | `./logs` | Directory for service stdout/stderr logs |

The service binds to `127.0.0.1` only. It is not reachable from the network.

---

## Security model

- **Auth:** Every request requires a Bearer token. Token is generated at install time and stored in `.env`.
- **Tier 1** tools are read-only and run unconditionally.
- **Tier 2** tools accept only a curated whitelist of sub-commands.
- **Tier 3** (`run_command`) defaults to `dry_run=true`. Claude must call twice ‚Äî once to preview, once to execute. Hard-blocked patterns reject the command regardless.
- **Audit log:** Every tool call is written to `logs/audit.log` with timestamp, tool name, arguments, and result tier.
- **Service protection:** `sc stop local-terminal-mcp` and `sc delete local-terminal-mcp` are hard-blocked, preventing Claude from accidentally killing its own connection.
